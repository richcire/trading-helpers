# 📘 트레이딩 계산 웹서비스 — 서비스 정의 & 기능 명세서 v1.0

> **목적**: 트레이더가 매번 머릿속으로 하거나 엑셀에서 하던 계산들을 하나의 웹페이지에서 빠르고 정확하게 처리한다.  
> **범위**: 프론트엔드 전용 (로그인/DB 없음, 브라우저만으로 동작)

---

## 0. 서비스 개요

### 0.1 왜 만드는가

트레이딩에서 실수가 가장 많이 나는 구간은 "계산"이다.

- 분할매수 후 평균단가가 얼마인지 순간 헷갈린다
- 손절가/익절가를 평균단가 기준으로 다시 세팅하는 게 번거롭다
- 수수료를 포함한 실제 손익이 표시되는 곳이 없다
- "감"으로 포지션 크기를 잡다가 손절에 계좌 큰 타격을 입는다
- 전략이 장기적으로 수익이 나는지 검증이 안 된다

이 서비스는 위 문제들을 **하나의 페이지에서 즉시 해결**하는 것을 목표로 한다.

### 0.2 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **프론트엔드 전용** | 서버 없음, 계산은 모두 브라우저에서 실행 |
| **설정 영속성** | LocalStorage로 전역 설정(수수료/레버리지 등)을 저장, 재방문 시 유지 |
| **롱/숏 모두 지원** | 현물(롱)뿐 아니라 선물 숏 포지션도 정확히 계산 |
| **수수료 현실 반영** | 진입/청산 수수료를 손익 및 손절/익절가에 반영하는 옵션 제공 |
| **즉시 계산** | 입력값 변경 시 실시간 계산 결과 반영 |

## 1. 전역 설정 (Global Settings)

> 모든 기능에 공통으로 적용되는 설정값. 헤더 또는 사이드패널에 항상 노출한다.

### 1.1 설정 필드

| 필드 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `leverage` | number | 1 | 레버리지 배수 (1 = 현물) |
| `feeEntryPct` | number | 0 | 진입 수수료 (% 단위, 예: 0.05) |
| `feeExitPct` | number | 0 | 청산 수수료 (% 단위) |
| `includeFeesInPnL` | boolean | true | 손익 계산 시 수수료 반영 여부 |
| `adjustStopTakePriceForFees` | boolean | false | 손절/익절 **가격 자체**를 수수료 보정해서 목표 수익률을 정확히 맞출지 여부 |

### 1.2 설정 정책 (필수)

- `adjustStopTakePriceForFees = ON`이면 `includeFeesInPnL`은 자동으로 ON 강제 (모순 방지)
- 모든 설정은 **LocalStorage에 자동 저장**되어 재방문 시 유지
- 레버리지는 **증거금(Margin)과 ROI 계산에만 영향**을 줌. 손익 금액 자체에는 영향 없음

## 2. 공통 계산식

> 모든 기능이 아래 수식을 공유한다.

### 2.1 용어 정의

| 기호 | 의미 |
|------|------|
| `E` | 평균단가 (진입 기준가) |
| `P` | 청산가 (손절/익절/시뮬레이션 가격) |
| `Q` | 총 수량 (항상 양수) |
| `L` | 레버리지 |
| `f_in` | 진입 수수료율 (소수, 예: 0.0005) |
| `f_out` | 청산 수수료율 (소수) |
| `s` | 방향 부호: LONG = +1, SHORT = -1 |
| `r` | 목표 수익률 (소수, 양수 = 수익, 음수 = 손실) |

### 2.2 수수료 미반영 손익

```
PnL_raw = s × (P - E) × Q
```
- LONG: `(P - E) × Q`
- SHORT: `(E - P) × Q`

### 2.3 수수료 반영 손익

**LONG:**
```
PnL_fee = P × Q × (1 - f_out) - E × Q × (1 + f_in)
```

**SHORT:**
```
PnL_fee = E × Q × (1 - f_in) - P × Q × (1 + f_out)
```

### 2.4 손익률 (PnL %)

```
LONG  분모: E × Q × (1 + f_in)
SHORT 분모: E × Q × (1 - f_in)
```
> 수수료 OFF일 때는 분모 = `E × Q`

### 2.5 레버리지 / ROI

```
Notional = E × Q
Margin   = Notional / L
ROI      = PnL / Margin
```

## 3. 기능 1 — 분할매수 평균단가 + 손절/익절 계산

### 3.1 목적

분할매수 후 평균단가를 자동 계산하고,  
그 기준으로 손절/익절 가격 및 실제 손익을 즉시 확인한다.

**사용자가 얻고 싶은 답:**
- "내 평균단가는 얼마지?"
- "손절 -3%면 가격이 어디야? (롱/숏 둘 다)"
- "그 가격에서 실제 손실은 수수료 포함 얼마지?"
- "최대 손실 10만원이면 손절가가 어디야?"

---

### 3.2 입력

| 입력 | 설명 |
|------|------|
| `direction` | LONG / SHORT 선택 |
| `entries[]` | 분할 매수 내역: 각 행마다 가격 + (수량 또는 금액) |
| `stopMode` | 손절 설정 방식: **% 기반** 또는 **금액 기반** |
| `stopValue` | 손절 값 (% 입력 또는 손실 허용 금액) |
| `takeMode` | 익절 설정 방식: **% 기반** 또는 **금액 기반** |
| `takeValue` | 익절 값 |
| `currentPrice` | 현재가 (입력 선택, 현재 손익 확인용) |

**입력 행 구성:**
- 가격 (필수)
- 수량 또는 금액 중 하나 선택 (행 모드 통일 또는 행별 토글)

---

### 3.3 평균단가 계산

```
E = Σ(price_i × qty_i) / Σ(qty_i)
Q = Σ(qty_i)
```

금액 모드 행: `qty_i = amount_i / price_i` 로 변환 후 동일 계산

---

### 3.4 손절/익절 가격 계산

#### % 기반 (수수료 보정 OFF)

```
P = E × (1 + s × r)
```
- LONG 손절 3%: `P = E × (1 - 0.03)`
- SHORT 익절 3%: `P = E × (1 - 0.03)` (가격 하락이 수익)

#### % 기반 (수수료 보정 ON)

**LONG:**
```
P = E × (1 + r) × (1 + f_in) / (1 - f_out)
```

**SHORT:**
```
P = E × (1 - r) × (1 - f_in) / (1 + f_out)
```

#### 금액 기반 (수수료 보정 OFF)

```
P = E + targetPnl / (s × Q)
```

#### 금액 기반 (수수료 보정 ON)

**LONG:**
```
P = (targetPnl + E × Q × (1 + f_in)) / (Q × (1 - f_out))
```

**SHORT:**
```
P = (E × Q × (1 - f_in) - targetPnl) / (Q × (1 + f_out))
```

---

### 3.5 출력

| 항목 | 설명 |
|------|------|
| 평균단가 `E` | |
| 총수량 `Q` | |
| 총 진입금액 | `E × Q` |
| 손절가 `P_stop` | |
| 손절 손익 (금액) | |
| 손절 손익률 (%) | |
| 익절가 `P_take` | |
| 익절 손익 (금액) | |
| 익절 손익률 (%) | |
| 현재가 기준 손익 | currentPrice 입력 시 |
| ROI | leverage > 1일 때 추가 표시 |

## 4. 기능 2 — DCA (물타기) 계산기

### 4.1 목적

기존 포지션에 추가 매수를 할 때 평균단가 변화와 본전가를 빠르게 확인한다.

**사용자가 얻고 싶은 답:**
- "추가 매수하면 평균단가가 어디로 이동하지?"
- "수수료 포함 실제 본전 가격은?"
- "본전까지 몇 % 올라야(또는 내려야) 하지?"

---

### 4.2 입력

| 입력 | 설명 |
|------|------|
| `direction` | LONG / SHORT |
| `E_old`, `Q_old` | 기존 포지션 평균단가 + 수량 |
| `P_add`, `Q_add` (또는 금액) | 추가 매수 가격 + 수량 |
| `currentPrice` | 현재가 (선택, 본전까지 거리 표시용) |

---

### 4.3 계산

```
E_new = (E_old × Q_old + P_add × Q_add) / (Q_old + Q_add)
Q_new = Q_old + Q_add
```

**본전가 (수수료 OFF):**
```
P_be = E_new
```

**본전가 (수수료 ON):**
- LONG: `P_be = E_new × (1 + f_in) / (1 - f_out)`
- SHORT: `P_be = E_new × (1 - f_in) / (1 + f_out)`

---

### 4.4 출력

| 항목 |
|------|
| 새 평균단가 `E_new` |
| 총 수량 `Q_new` |
| 본전가 `P_be` |
| 본전까지 필요 이동률 (%) |
| 현재 평가손익 (currentPrice 있을 때) |

---

## 5. 기능 3 — 리스크 기반 포지션 사이징

### 5.1 목적

손절가를 정했을 때 계좌 리스크를 지키는 적정 수량을 계산한다.

**사용자가 얻고 싶은 답:**
- "계좌의 최대 1% 손실 기준으로 몇 개를 사야 하지?"
- "포지션 명목 금액과 증거금은 얼마?"

---

### 5.2 입력

| 입력 | 설명 |
|------|------|
| `direction` | LONG / SHORT |
| `accountEquity` | 계좌 총 자산 |
| `riskPct` 또는 `riskAmount` | 허용 손실 (% 또는 금액) |
| `entryPrice E` | 진입가 |
| `stopPrice P_stop` | 손절가 |

---

### 5.3 계산

```
riskAmount = accountEquity × riskPct / 100   (% 모드일 때)

lossPerUnit = max(0, -s × (P_stop - E))
  LONG:  lossPerUnit = E - P_stop
  SHORT: lossPerUnit = P_stop - E

Q = riskAmount / lossPerUnit
```

**수수료 반영 (보수적 사이징, 선택):**
- LONG Q=1 기준 손실: `E×(1+f_in) - P_stop×(1-f_out)`
- SHORT Q=1 기준 손실: `P_stop×(1+f_out) - E×(1-f_in)`

---

### 5.4 출력

| 항목 |
|------|
| 권장 수량 `Q` |
| 포지션 명목 `E × Q` |
| 증거금 `Margin` |
| 손절 시 손익 금액 |
| 손절 시 ROI (leverage > 1) |

---

## 6. 기능 4 — 포지션 시뮬레이터

### 6.1 목적

가격 변화에 따른 손익 곡선을 시각화하여 리스크 구조를 직관적으로 파악한다.

**사용자가 얻고 싶은 답:**
- "이 가격이면 손익이 얼마지?"
- "손절선/익절선이 손익 그래프에서 어디야?"
- "레버리지 쓰면 ROI가 어떻게 달라지나?"

---

### 6.2 입력

| 입력 | 설명 |
|------|------|
| `direction` | LONG / SHORT |
| `E`, `Q` | 평균단가, 수량 |
| `minPrice`, `maxPrice` | 시뮬레이션 가격 범위 |
| `steps` | 구간 수 (기본 200) |
| 손절가/익절가 | 그래프 위 마커 표시용 (선택) |

---

### 6.3 계산

```
각 price_i = minPrice + i × (maxPrice - minPrice) / (steps - 1)

PnL_i = PnL(price_i) // includeFeesInPnL에 따라 수식 선택
PnLPct_i = PnL_i / 분모
ROI_i = PnL_i / Margin  // leverage > 1일 때
```

---

### 6.4 출력 (차트)

- x축: 가격
- y축: 손익 금액 (선택: 손익률 / ROI 전환 가능)
- 손절가, 익절가, 평균단가, 현재가 수직선 표시
- 차트 위 마우스 호버 시 해당 가격에서의 손익 툴팁

---

## 7. 기능 5 — 기대값 & 손익비 분석

### 7.1 목적

전략의 장기 수익성을 승률과 손익비로 검증한다.

**사용자가 얻고 싶은 답:**
- "내 승률과 손익비로 장기적으로 수익이 나는가?"
- "손익분기 승률은 얼마인가?"
- "손익비를 높이면 필요 승률이 어떻게 변하나?"

---

### 7.2 입력

| 입력 | 설명 |
|------|------|
| `winRatePct` | 승률 (%) |
| `RR` | 손익비 (예: 2 = 수익이 손실의 2배) |
| `avgWin`, `avgLoss` | RR 대신 직접 금액 입력 (선택 모드) |

---

### 7.3 계산

```
p = winRatePct / 100
q = 1 - p

// RR 모드
Expectancy_R = p × RR - q × 1

// 금액 모드
Expectancy = p × avgWin - q × avgLoss

// 손익분기 승률
breakEvenWinRate = 1 / (1 + RR)
```

---

### 7.4 출력

| 항목 |
|------|
| 기대값 (R 배수 또는 금액) |
| 손익분기 승률 |
| 전략 판정: 양의 기대값 여부 |
| 손익비 vs 필요 승률 차트 (손익비 변화에 따른 BEP 커브) |

## 8. UX & UI 명세

### 8.1 페이지 구조

- 단일 페이지 (SPA) — 탭 또는 섹션으로 5개 기능 전환
- 상단 고정 헤더: 서비스명 + 전역 설정 버튼
- 전역 설정은 모달 또는 사이드 패널로 열림

### 8.2 실시간 계산

- 모든 입력 필드는 `onChange` 즉시 계산 → 별도 "계산하기" 버튼 없음
- 단, 분할매수 행 추가/삭제 시에도 즉시 재계산

### 8.3 입력 편의 기능

- 분할매수 행: **+행 추가 / 행 삭제** 버튼
- 수량/금액 **모드 전환** 토글 (행 단위 또는 전체 단위)
- 계산 결과 **클립보드 복사** 버튼 (결과 텍스트 복사)
- **초기화** 버튼 (현재 기능 입력값만 초기화)

### 8.4 결과 표시

- 수익/이익 숫자: **초록색**
- 손실 숫자: **빨간색**
- 숫자 포맷: 천 단위 콤마, 소수 자리 적절히 표시

### 8.5 반응형

- 모바일/태블릿/데스크탑 모두 대응 (트레이딩은 모바일에서도 많이 확인함)

### 8.6 설정 영속성

- 전역 설정(수수료, 레버리지 등)은 **LocalStorage에 자동 저장**
- 각 기능의 마지막 입력값도 선택적으로 LocalStorage 저장 (UX 개선)

## 9. 엣지 케이스 & 검증 규칙

### 9.1 공통 입력 유효성

| 조건 | 처리 |
|------|------|
| 가격 ≤ 0 | 에러 표시, 계산 중단 |
| 수량 ≤ 0 | 에러 표시, 계산 중단 |
| 레버리지 < 1 | 에러 표시 |
| 수수료 < 0 또는 ≥ 100% | 에러 표시 |
| `adjustStopTakePriceForFees=ON`이고 `f_out ≥ 1` | 분모 0 방지, 에러 |

### 9.2 평균단가

| 조건 | 처리 |
|------|------|
| 입력 행 0개 | "매수 내역을 입력하세요" 표시 |
| 총 수량 = 0 | 계산 불가 처리 |
| 금액 모드에서 가격 = 0 | 해당 행 무효 표시 |

### 9.3 손절/익절

| 조건 | 처리 |
|------|------|
| 손절% ≥ 100% | 에러 |
| 계산된 손절/익절가 ≤ 0 | 에러 |
| 익절 값 음수 입력 | 에러 |

### 9.4 포지션 사이징

| 조건 | 처리 |
|------|------|
| LONG인데 손절가 ≥ 진입가 | "손절가가 진입가보다 낮아야 합니다" |
| SHORT인데 손절가 ≤ 진입가 | "손절가가 진입가보다 높아야 합니다" |
| lossPerUnit = 0 | 사이징 불가 처리 |

### 9.5 시뮬레이터

| 조건 | 처리 |
|------|------|
| minPrice ≥ maxPrice | 자동 스왑 또는 에러 |
| steps < 10 또는 > 1000 | 10~1000 범위로 클램프 |
